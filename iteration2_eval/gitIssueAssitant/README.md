# 迭代二
项目目标
实现一个面向 GitHub Issue 的智能修复 Agent，使其能够围绕真实 issue，自主完成任务理解、修复规划、工具调用、代码修改、测试验证、失败后自我修正，并将全过程进行可视化展示。
________________________________________
分组方案
每组 2 人。
第一组：Agent 主链 + 推理策略组
职责
•	设计统一的 AgentState 
•	实现主 orchestrator，串联整体流程 
•	实现 planning、ReAct、Reflexion 
•	定义成功、失败和终止条件 
•	维护主链状态流转 
交付
•	AgentState 数据结构 
•	orchestrator 模块 
•	planner / ReAct / Reflexion 模块 
•	成功/失败判定逻辑 
•	主链流程与状态接口说明 
________________________________________
第二组：工具链 + 执行环境组
职责
•	实现 Read_File、Code_Search、Patch_File 
•	实现 Bash_Terminal 和 pytest runner 
•	基于 LangChain `@tool` 统一工具声明与调用 
•	设计执行环境约束 
交付
•	文件读取、代码搜索、代码修改、终端执行、测试运行工具 
•	LangChain 工具定义与 `ToolNode` 接入
•	patch diff 生成逻辑
•	工具使用说明
•	执行环境说明 
________________________________________
第三组：可视化 + 工程保障组
职责
•		CLI / Web 展示界面
•	设计并实现状态流转展示 
•	实现 GitHub Issue 接入 
•	搭建 CI/CD 
•	编写运行、部署和演示文档与演示视频
•	状态流转展示模块 
•	GitHub Issue 接入模块 
•	Dockerfile / 启动脚本 
•	CI/CD 配置 
•	项目运行、部署、演示文档


三组之间的总体关联
可以把三组关系理解成这条链：
第一组：决定下一步做什么
   ↓
第二组：把这一步真正执行出来
   ↓
第一组：根据执行结果继续判断
   ↓
第三组：把整个过程展示出来，并保证系统能稳定运行
