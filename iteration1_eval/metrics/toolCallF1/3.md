# Tool Call F1 (工具调用F1分数)

ToolCallF1 是一个用于评估 AI Agent 调用外部工具准确性的指标。通过对比 Agent 实际执行的工具调用轨迹（Actual Steps）与期望的工具调用轨迹（Expected Steps），计算两者的集合匹配情况，最终输出基于精确率（Precision）和召回率（Recall）的 F1 分数。

## 1. 评分逻辑 (Scoring Logic)

本指标使用项目内置的 LLM（`SimpleLLM`）作为裁判来进行语义级别的相似度匹配。
匹配规则如下：
1. `tool_call` 名称必须具有一一对应关系或完全匹配。
2. `input`（即工具传入参数）在语义上是等价且能完成相同目标的。即使某些语法不同（如缺少可选参数，字典键序打乱），只要实现效果等价即认为匹配成功。

**评价指标计算方式**：
- **True Positives (TP)**: 期望轨迹与实际轨迹成功匹配的工具调用数量。
- **False Positives (FP)**: 实际调用中多余（未在期望中匹配上）的工具调用数量。
- **False Negatives (FN)**: 期望调用中缺失（未在实际调用中体现）的工具调用数量。

- **Precision (精确率)** = TP / (TP + FP)
- **Recall (召回率)** = TP / (TP + FN)
- **F1 Score** = 2 * (Precision * Recall) / (Precision + Recall)

最终返回值为 `0.0` 到 `1.0` 之间的浮点数，以及包含 TP、FP、FN 及匹配细节的字典对象。

## 2. 输入要求 (Input Requirements)

在评估框架传递给本指标的 `sample`（单个测试样本字典格式）中，必须包含如下字段：
- `expected_steps` (List[Dict]): 标准答案所需的工具调用列表。每个步骤需包含 `tool_call`（字符串）和 `input`（参数字符串）。
- `steps` (List[Dict]): 模型实际生成的工具调用列表。结构需同 `expected_steps` 保持一致。

**输入样例 (Input Example)**:
```json
{
  "expected_steps": [
    {
      "step": 1,
      "tool_call": "search_restaurants",
      "input": "{\"location\": \"仙林校区\", \"restaurant_name\": \"张记烤肉店\"}"
    }
  ],
  "steps": [
    {
      "step": 1,
      "tool_call": "search_restaurants",
      "input": "{\"location\": \"仙林校区\"}"
    }
  ]
}
```

## 3. 使用方法 (Usage)

在您的 `run_eval.py` 评估脚本中，按照如下步骤注册、加载并执行本指标：

```python
import importlib.util
from llm import SimpleLLM
from evaluator import Evaluator

# 1. 设置指标存储路径
metric_path_toolCallF1 = Path(__file__).parent / "metrics" / "3" / "metric.py"

# 2. 调用内置的 load_metric 进行动态加载
metric_ToolCallF1 = load_metric(metric_path_toolCallF1)

# 3. 将指标添加至 Evaluator 的 metrics 列表中以执行评估
evaluator = Evaluator(metrics=[metric_ToolCallF1], llm=SimpleLLM())
results = evaluator.evaluate(dataset)

# 4. 打印或获取最终得分
for index, result in enumerate(results, start=1):
    f1_score = result.get("tool_call_f1")["score"]
    details = result.get("tool_call_f1")["reasoning"]
    print(f"Sample {index} - ToolCall F1 Score: {f1_score}")
```
