# Topic Adherence

`metrics/1` 复现的是简化版 `TopicAdherence`。

## 当前支持的样本格式

标准格式：

```json
{
  "reference_topics": ["Python", "programming"],
  "messages": [
    {"role": "human", "content": "Talk about Python and cooking."},
    {"role": "ai", "content": "Python is a programming language. I can't help with cooking."}
  ]
}
```

`eval_dataset.json` 格式：

```json
{
  "task_id": "42966ceff3ff344b",
  "user_query": "帮我点一份双人套餐",
  "final_answer": "已经为您下单成功"
}
```

当输入是 `eval_dataset.json` 结构时，指标会自动：

1. 用 `user_query` 构造 Human 消息
2. 用 `final_answer` 构造 AI 消息
3. 如果没有 `reference_topics`，就从 `user_query` 自动抽 topic

## 评分逻辑

与 ragas 原版一致的骨架是：

1. 抽取 topic
2. 判断 topic 是否被回答或拒答
3. 判断 topic 是否属于允许主题
4. 计算 `precision / recall / f1`

当前是规则法实现，不调用 ragas 顶层 `evaluate()`。
