### 分层说明

| 层级 | 主要职责 | 关键文件 |
| --- | --- | --- |
| 前端层 | 创建任务、展示执行过程、提交补充指令、查看 diff/trace、发起 Push/PR/Issue 写回 | `frontend/src/App.tsx`, `frontend/src/pages/*`, `frontend/src/services/api.ts` |
| REST API 适配层 | 将 HTTP 请求转发到核心服务，统一响应模型和错误码 | `gitIssueAssitant/RESTAPIAdapter/main.py`, `api/routes/*.py` |
| 应用服务层 | 任务生命周期、运行时构建、结果同步、Trace、配置、Skill、GitHub Issue | `gitIssueAssitant/core/services/*` |
| Agent 运行层 | LLM 规划、ReAct 推理、工具执行、安全拦截、沙箱执行、上下文压缩 | `gitIssueAssitant/core/agent/*` |
| 数据与外部依赖 | SQLite 持久化、`.env` 配置、Skill 文件、本地仓库、Docker、GitHub、模型 API | `.agent_data`, `.env`, `repos`, `workspaces` |

## 2. 系统流程图：任务创建与自动执行

```mermaid
sequenceDiagram
    autonumber
    actor User as 用户
    participant FE as React 前端
    participant API as FastAPI 路由
    participant Facade as IssueAssistantService
    participant TaskSvc as TaskService
    participant Runtime as _IssueAssistantTaskRuntime
    participant Session as SessionService
    participant Agent as AgentOrchestrator
    participant Tool as ToolRegistry / AGENT_TOOLS
    participant Store as SQLite / StateStore
    participant External as OpenAI / GitHub / Docker / Git

    User->>FE: 填写仓库、Issue、模型、Skill 并创建任务
    FE->>API: POST /api/tasks/
    API->>Facade: create_task(TaskCreate)
    Facade->>TaskSvc: create_task(payload)
    TaskSvc->>Store: 保存 TaskRecord(status=draft)
    Store-->>TaskSvc: TaskRecord
    TaskSvc-->>Facade: TaskResponse

    alt auto_start = true
        API->>Facade: run_task(task_id, mode=auto, reset=true)
        Facade->>Runtime: run(task_id, TaskRunRequest)
        Runtime->>Store: 读取 TaskRecord
        Runtime->>Runtime: 注册后台 job，串行执行
        Runtime->>Runtime: 加载 .env 与启用的 Skill
        Runtime->>Facade: create_for_rest_runtime(...)
        Facade->>External: 初始化 ChatOpenAI
        Facade-->>Runtime: 带 Agent/工具/沙箱的运行时 Facade
        Runtime->>Session: create_session(repo_source, target_dir)
        Session->>External: git clone 或复用本地仓库
        Session->>External: 解析 GitHub Issue 文本（可选）
        Session->>External: 启动 DockerSandbox（可选）

        alt Docker 沙箱不可用且未选择本地回退
            Runtime->>Store: 保存 SANDBOX_UNAVAILABLE 状态
            Runtime-->>API: 返回 scheduled，等待用户选择
        else 运行环境就绪
            Session->>Store: 初始化并持久化 LangGraph state
            Runtime->>Agent: stream_graph_updates(thread_id)
            loop LangGraph 自动循环
                Agent->>Agent: h_planner 选择 Skill 并生成目标
                Agent->>External: 调用 LLM 生成 ReAct 决策
                Agent->>Tool: 检查并执行工具调用
                Tool->>External: 文件读写、搜索、测试、Git、Shell
                Tool-->>Agent: ToolMessage + ToolCallEvent
                Agent->>Agent: reflect 或继续 react
                Runtime->>Store: 同步 timeline、messages、metrics、trace
            end
            Agent-->>Runtime: SUCCESS 或 FAILED
            Runtime->>Store: 刷新 TaskResult、AgentTrace、FixReport
        end
    end

    API-->>FE: TaskResponse
    FE->>API: 定时 GET /api/tasks/ 与 /api/tasks/{id}/messages
    API-->>FE: 最新任务状态、对话与结果
    User->>FE: 查看 diff / trace / 报告，必要时 Push、创建 PR 或写回 Issue
```

## 3. Agent 内部状态流图

```mermaid
flowchart LR
    Start([开始]) --> Planner["h_planner<br/>选择 Skill、生成或更新目标"]
    Planner -->|还有未完成目标| React["react<br/>LLM ReAct 推理"]
    Planner -->|目标全部完成| Success([finish_success])

    React -->|发起工具调用| Tools["tools<br/>ToolRegistry 校验<br/>执行 AGENT_TOOLS"]
    Tools --> React

    React -->|GOAL_DONE 或近期成功编辑| Planner
    React -->|TASK_SUCCESS| Success
    React -->|TASK_FAILED| Failed([finish_failed])
    React -->|连续无进展或重复回复| Failed
    React -->|无工具调用且未终止| Reflect["reflect<br/>分析失败轨迹"]

    Reflect -->|需要重规划且 plan_version &lt; 3| Planner
    Reflect -->|继续尝试| React

    UserMsg["用户追加要求<br/>submit_task_message"] -. inject_message .-> React
    UserMsg -. reopen_after_terminal .-> Reopen["reopen<br/>终态任务重新打开"]
    Reopen -->|有 replan_trigger| Planner
    Reopen -->|继续当前上下文| React
```

## 6. 关键状态流转

```mermaid
stateDiagram-v2
    [*] --> draft: 创建任务
    draft --> scheduled: auto_start 或手动运行
    scheduled --> running: 后台 job 开始执行
    running --> scheduled: Docker 沙箱不可用，等待用户选择
    scheduled --> running: 用户选择本地回退或重新运行
    running --> completed: Agent 输出 TASK_SUCCESS
    running --> failed: Agent 输出 TASK_FAILED 或运行异常
    completed --> scheduled: 用户追加要求并重新打开任务
    failed --> scheduled: 用户追加要求或重置重跑
    completed --> [*]
    failed --> [*]
```

## 7. 主要接口清单

| 模块 | 接口 | 用途 |
| --- | --- | --- |
| 健康检查 | `GET /health` | 前端判断后端是否在线 |
| 任务 | `GET /api/tasks/` | 获取任务列表 |
| 任务 | `POST /api/tasks/` | 创建任务，可自动启动 |
| 任务 | `POST /api/tasks/{task_id}/run` | 运行、重跑或本地回退执行任务 |
| 任务 | `GET/POST /api/tasks/{task_id}/messages` | 获取对话、提交补充要求 |
| 任务 | `GET /api/tasks/{task_id}/diff` | 获取本地仓库变更 |
| 任务 | `GET /api/tasks/{task_id}/trace` | 获取标准 AgentTrace |
| 任务 | `GET /api/tasks/{task_id}/report` | 下载修复报告 |
| 发布 | `POST /api/tasks/{task_id}/push` | 提交并推送任务变更 |
| 发布 | `POST /api/tasks/{task_id}/pull-request` | 创建 Pull Request |
| GitHub Issue | `GET /api/repos/issues` | 浏览仓库 Issues |
| GitHub Issue | `GET /api/tasks/{task_id}/issue` | 获取任务关联 Issue |
| GitHub Issue | `POST /api/tasks/{task_id}/issue/comment` | 写回 Issue 评论 |
| GitHub Issue | `PATCH /api/tasks/{task_id}/issue/state` | 关闭 Issue |
| 评测对比 | `GET /api/analytics/compare` | 对比多个任务指标 |
| 评测对比 | `GET /api/analytics/report.md` / `report.csv` | 导出对比报告 |
| Skill | `GET/POST /api/skills/` | 查看或创建 Skill |
| Skill | `PUT /api/skills/{name}/enabled` | 启用或禁用 Skill |
| 设置 | `GET/PUT /api/settings/` | 读取或更新 `.env` 配置 |
| 设置 | `GET /api/settings/models` | 从模型 API 拉取模型列表 |
