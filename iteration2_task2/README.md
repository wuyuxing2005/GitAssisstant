目标：搭建一个Agent应用评估平台
基本约束：前后端分离Web项目，开发语言不要求，评估指标可以直接使用Ragas库
基本需求点：
评测任务管理能力：支持创建评测任务，支持配置评测任务（如Agent版本、数据集选择、评测方法选择等），支持评测任务增删改查和状
态管理与展示
评测执行与方法能力：支持多种评测方法（三种划分不独立，相互间必然有重叠）
按评估模式划分：面向结果（只关注输入输出）/ 面向过程（关注 Agent 响应的中间过程）
按评估方式划分：显式指标（可以明确计算，如 Token 消耗、工具调用正确率、任务成功率、响应时间等）/ 模糊指标（评价标准相对主观，一般以 LLM-as-
a-Judge 的模式实现，例如推理质量、输出内容准确性、幻觉程度、交互体验等，可以自行设计，也可以基于已有评估框架实现）
按评估维度划分：效果维度（任务响应是否有效、准确、完整）/ 安全（反馈信息是否安全，是否不包含有害内容）/ 性能（用户体验是否流畅，是否出现长
时间停顿等待，是否有良好的互动交流体验）
扩展能力：支持自定义评估指标，支持组合评估策略
评测分析与展示能力：支持单次评估任务可视化结果展示，支持多个评估任务间对比分析与展示
